## 一、项目背景与动机

### 1.1 缘起

[OpenClaw](https://github.com/openclaw/openclaw) 是一个优秀的开源 AI 个人助手项目，基于 Node.js 构建，支持多平台消息接入、工具调用、技能系统等。但其安装依赖 Node.js + npm 运行时环境，配置流程繁琐，对非技术用户门槛极高。

**核心痛点**：一个普通用户想用 AI 操控电脑，却要先学 npm install。

### 1.2 目标

RS-Claw 的目标是**用 Rust 重写 OpenClaw 的核心能力**，实现：

1. **零依赖分发** — 编译为单一二进制文件，不需要 Node.js、Python 或任何运行时
2. **双击即用** — 配置一次 API Key，像聊天一样使用
3. **最低门槛** — 同时服务开发者（Gateway API）和普通用户（CLI 对话）
4. **跨平台** — Linux x86_64 优先，Windows/macOS 跟进

## 二、架构设计

### 2.1 总体架构

```
┌──────────┐     ┌───────────┐     ┌──────────┐
│   CLI    │────▶│  Gateway   │────▶│  Agent   │
│  (REPL)  │     │ (WS JSON) │     │ Runtime  │
└──────────┘     └───────────┘     └────┬─────┘
                                        │
                              ┌─────────┼─────────┐
                              │         │         │
                          ┌───▼──┐ ┌───▼──┐ ┌───▼──┐
                          │Model │ │Tools │ │Memory│
                          │Layer │ │ L1-L3│ │System│
                          └──────┘ └──────┘ └──────┘
```

核心思想沿用 OpenClaw 的 **Gateway Hub** 模式——通过 WebSocket JSON-RPC 统一消息入口，CLI/外部程序均可通过同一协议接入 Agent 运行时。

### 2.2 能力分层（5 层工具系统）

| 层级 | 能力 | 优先级 | 状态 |
|:---|:---|:---|:---|
| **L1** 文件系统 | 读、写、列目录、检查存在 | MVP | ✅ 已完成 |
| **L2** 进程管理 | 执行 Shell 命令 | MVP | ✅ 已完成 |
| **L3** 网络请求 | HTTP GET / POST | MVP | ✅ 已完成 |
| **L4** 桌面操控 | AT-SPI2 无障碍 + 视觉模型兜底 | Phase 3 | 🚧 骨架 |
| **L5** 浏览器操控 | CDP 协议 / chromiumoxide | Phase 4 | 🚧 骨架 |

### 2.4 通信协议

Gateway 使用 **WebSocket JSON-RPC 帧协议**：

```
请求:  {"type":"req",  "id":"1", "method":"chat.send", "params":{...}}
响应:  {"type":"res",  "id":"1", "ok":true, "payload":{...}}
事件:  {"type":"event","event":"agent","payload":{"stream":"assistant","data":{...}}}
```

支持 `chat.send`、`health` 等方法，支持流式事件推送 `assistant` / `tool` / `error`。

### 2.5 配置系统

配置文件位置：`~/.rs-claw/config.toml`

```toml
[gateway]    # REST + WebSocket 端口
[model]      # provider / api_key / model / base_url (支持 ${ENV} 引用)
[vision]     # 视觉模型 + 无障碍回退 + 坐标模式
[memory]     # max_session_messages / compaction_threshold / auto_compaction
[skill]      # auto_refine / similarity_threshold
```

## 六、技术栈总结

| 组件 | 选型 | 理由 |
|:---|:---|:---|
| 语言 | Rust 2021 | 零成本抽象、内存安全、编译为单文件 |
| 异步运行时 | Tokio | Rust 生态标准 |
| HTTP 客户端 | reqwest (rustls) | 支持 streaming、JSON |
| WebSocket | tokio-tungstenite | Gateway 核心 |
| CLI | clap 4 derive | 声明式命令行 |
| 序列化 | serde + toml | 配置 + JSON-RPC 协议 |
| 数据库 | rusqlite (bundled) | 免安装 SQLite |
| 向量检索 | usearch (HNSW) | 轻量高性能 |
| 日志 | tracing + tracing-subscriber | 结构化日志 |
| LLM 接口 | OpenAI 兼容协议 | 一套格式兼容多个模型商 |
