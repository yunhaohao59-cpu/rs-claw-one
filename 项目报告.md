## 一、项目背景与动机

### 1.1 缘起

[OpenClaw](https://github.com/openclaw/openclaw) 是一个优秀的开源 AI 个人助手项目，基于 Node.js 构建，支持多平台消息接入、工具调用、技能系统等。但其安装依赖 Node.js + npm 运行时环境，配置流程繁琐，对非技术用户门槛极高。

**核心痛点**：一个普通用户想用 AI 操控电脑，却要先学 `npm install`。

### 1.2 目标

RS-Claw 的目标是**用 Rust 重写 OpenClaw 的核心能力**，实现：

1. **零依赖分发** — 编译为单一二进制文件，不需要 Node.js、Python 或任何运行时
2. **双击即用** — 配置一次 API Key，像聊天一样使用
3. **最低门槛** — 同时服务开发者（Gateway API）和普通用户（CLI 对话）
4. **跨平台** — Linux x64 / ARM64 + Windows x64 已验证，macOS 待补

## 二、架构设计

### 2.1 总体架构

```
┌──────────┐     ┌───────────┐     ┌──────────┐
│   CLI    │────▶│  Gateway   │────▶│  Agent   │
│  (REPL)  │     │ (WS JSON) │     │ Runtime  │
└──────────┘     └───────────┘     └────┬─────┘
                                        │
                              ┌─────────┼─────────┐
                              │         │         │
                          ┌───▼──┐ ┌───▼──┐ ┌───▼──┐
                          │Model │ │Tools │ │Memory│
                          │Layer │ │ L1-L3│ │System│
                          └──────┘ └──────┘ └──────┘
```

核心思想沿用 OpenClaw 的 **Gateway Hub** 模式——通过 WebSocket JSON-RPC 统一消息入口，CLI/外部程序均可接入 Agent。

Agent 运行时内部流程：

```
用户消息 → System Prompt + 工具定义 + 记忆注入 + 对话历史
         → LLM 推理
         → 有 tool_calls? → 调度工具 → 结果注入历史 → 回到 LLM (max 10轮)
         → 无 tool_calls? → 返回最终回复
                        → 自动提炼 Skill → 存入 SQLite + VectorStore
```

### 2.4 通信协议

WebSocket JSON-RPC 帧协议：

```
请求:  {"type":"req",  "id":"1", "method":"chat.send", "params":{...}}
响应:  {"type":"res",  "id":"1", "ok":true, "payload":{...}}
事件:  {"type":"event","event":"agent","payload":{"stream":"assistant","data":{...}}}
```

### 2.5 配置系统

`~/.rs-claw/config.toml`，支持 `${ENV}` 环境变量引用：

```toml
[gateway]    # WebSocket 端口
[model]      # provider / api_key / model / base_url
[memory]     # max_session_messages / compaction_threshold / auto_compaction / top_k_memories
[skill]      # auto_refine / similarity_threshold
```

## 六、技术栈

| 组件 | 选型 | 理由 |
|:---|:---|:---|
| 语言 | Rust 2021 | 零成本抽象、单文件编译 |
| 异步运行时 | Tokio | 生态标准 |
| HTTP 客户端 | reqwest (rustls) | streaming + JSON |
| WebSocket | tokio-tungstenite | Gateway 核心 |
| CLI | clap 4 | 声明式命令行 |
| 序列化 | serde + toml | 配置 + JSON-RPC |
| 数据库 | rusqlite (bundled) | 免安装 SQLite |
| 向量检索 | trigram embedding | 256维、文件级存储 |
| 跨平台编译 | mingw-w64 + zig | Win/Linux ARM64 |
| 日志 | tracing | 结构化日志 |
